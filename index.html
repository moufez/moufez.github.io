Web Attack Surface

OWASP TOP 10 // 2025

01 A01:2025
Broken Access Control

Access control enforces policy that prevents users from acting outside their intended permissions. Failures lead to unauthorized data access, modification, or deletion.

IDOR — manipulating object references (user IDs, file paths) to access unauthorized resources. Confirmed real-world finding.
02 A02:2025
Security Misconfiguration

Default credentials, unnecessary features enabled, verbose error messages, missing security headers — the most frequently encountered class in real environments.

Exposed .git directories, default admin credentials, directory listing, verbose stack traces revealing internal paths.
03 A03:2025
Software Supply Chain Failures

Using components with known vulnerabilities, outdated libraries, or unverified third-party dependencies. Attack surface extends beyond your own code.

Outdated CMS plugins with public CVEs, vulnerable JavaScript libraries loaded via CDN, unpatched server-side dependencies.
04 A04:2025
Cryptographic Failures

Data transmitted or stored without adequate encryption. Weak algorithms, hardcoded keys, improper certificate validation all fall here.

HTTP transmitting credentials in plaintext, MD5/SHA1 password hashes, hardcoded API keys in JavaScript source.
05 A05:2025
Injection

Hostile data is sent to an interpreter as part of a command or query. SQL, OS command, LDAP, and template injection remain highly impactful.

SQLi via unsanitized input — UNION-based data extraction, error-based extraction, blind boolean/time-based enumeration.
06 A06:2025
Insecure Design

Design flaws distinct from implementation bugs. Missing rate limits, predictable resource locations, lack of threat modeling baked into architecture.

Sequential resource IDs enabling enumeration, password reset flows with guessable tokens, missing account lockout mechanisms.
07 A07:2025
Authentication Failures

Weaknesses in authentication and session management allow attackers to impersonate legitimate users or bypass login controls entirely.

JWT algorithm confusion (alg:none), weak session tokens, credential stuffing, account takeover via broken password reset.
08 A08:2025
Software & Data Integrity Failures

Code and infrastructure that doesn't protect against integrity violations — insecure deserialization, unsigned auto-updates, unvalidated CI/CD pipelines.

Insecure deserialization of serialized objects (PHP/Java), tampered JWT payloads without signature verification.
09 A09:2025
Security Logging & Alerting Failures

Insufficient logging, monitoring, and alerting means breaches go undetected. Attackers rely on blind spots to maintain persistence unnoticed.

Brute force attacks with no lockout or alerting, failed login attempts not logged, no anomaly detection on sensitive endpoints.
10 A10:2025
Mishandling Exceptional Conditions

Improper error handling exposes internal state, stack traces, and application logic that assists attackers in understanding the target architecture.

Verbose SQL errors revealing database schema, unhandled exceptions disclosing server paths, debug mode left enabled in production.

VULNERABILITY REPORT // FINDING #001
INSECURE DIRECT OBJECT REFERENCE (IDOR) — HORIZONTAL PRIVILEGE ESCALATION
Classification: CONFIDENTIAL
Severity: HIGH
CVSS: 7.5 (AV:N/AC:L/PR:L/UI:N)
Vulnerability
Insecure Direct Object Reference (IDOR) — Broken Access Control
Severity
HIGH
Affected
/api/user/profile?id=[REDACTED]
Description
The application exposes user account data via a predictable numeric ID parameter. The server performs no authorization check to verify that the requesting user owns the referenced resource. Any authenticated user can retrieve full account details of other users by modifying the ID value.
PoC
# Legitimate request (own profile) GET /api/user/profile?id=1042 HTTP/1.1 Authorization: Bearer eyJ[...REDACTED...] # Modified — targeting another user GET /api/user/profile?id=1043 HTTP/1.1 Authorization: Bearer eyJ[...REDACTED...] HTTP/1.1 200 OK {"id": 1043, "name": "[REDACTED]"}
Impact
Unauthorized access to PII for all registered users. An attacker can enumerate and extract full user records at scale.
Remediation
Implement server-side authorization on every resource request. Verify session user ID matches requested resource owner. Use UUIDs instead of sequential numeric IDs. Apply principle of least privilege.
